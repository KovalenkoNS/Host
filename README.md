# scg-host — управляющее приложение SCG

`scg-host` — это «пульт управления»: небольшая программа, которая **только
запускает приложения и показывает их результаты**. Сама она ничего не
вычисляет.

Приложение здесь — это **проект**: папка с одним или несколькими
исполняемыми файлами (`.exe`). Проект подключается из одного из двух
источников:

* **локальный проект** — папка на вашем компьютере (используется на месте,
  ничего не копируется);
* **проект с GitHub** — публичный репозиторий, скачиваемый целиком по одной
  ссылке в папку `external-apps/`.

В проекте может быть несколько `.exe`. Хост показывает их списком, а вы
выбираете нужный и запускаете его как есть: каждый запуск — отдельный процесс
операционной системы, у которого хост забирает код выхода и вывод (stdout).

Сводка о подключённых проектах сохраняется в файл `scg-apps.json` и
обновляется автоматически при любом изменении.

## 1. Для кого этот документ

Разделы 2–4 написаны для людей **без опыта программирования**: как получить
программу, запустить её и пользоваться через браузер. Разделы 5–8 — для
инженеров и разработчиков.

## 2. Установка и запуск (для неспециалистов)

### Шаг 1. Один раз установить Go (это «сборщик» программы)

1. Откройте <https://go.dev/dl/> и скачайте установщик для вашей системы
   (Windows — файл `.msi`, «Далее → Готово»).
2. Проверка: в командной строке введите `go version` — должна появиться
   строка вида `go version go1.22...`.

### Шаг 2. Распаковать архив

Распакуйте `scg-host` в любую папку. Внутри должны быть `cmd`, `internal`
и файлы `go.mod`, `Makefile`, `README.md`.

### Шаг 3. Собрать программу (одна команда, один раз)

В командной строке **в папке программы** выполните:

    go build -o bin/scg-host ./cmd/scg-host

*(в Windows: `go build -o bin\scg-host.exe .\cmd\scg-host`)*

Появится папка `bin` с готовой программой. Интернет для сборки не нужен.

### Шаг 4. Запустить

Запустите программу **без параметров** — двойным щелчком по
`bin\scg-host.exe` (Windows) или командой `bin/scg-host`. Программа
запустится на вашем компьютере и **сама откроет браузер** на нужной странице
(обычно http://localhost:8080). Окно не закрывайте — пока оно открыто, всё
работает. Остановка — `Ctrl+C`.

Особые случаи, которые программа решает сама:

* scg-host **уже запущен** (щёлкнули по exe второй раз) — новый экземпляр не
  поднимается, открывается страница уже работающего;
* порт 8080 **занят** — берётся следующий свободный (8081, 8082, …),
  фактический адрес печатается в окне.

Рядом с программой создаются папка `external-apps/` (для скачанных проектов)
и файл `scg-apps.json` (сводка подключённых проектов).

## 3. Как пользоваться (веб-страница)

На странице — две формы подключения, список проектов и таблица заданий.

* **Форма «Подключить локальный проект»** — впишите путь к папке проекта на
  вашем компьютере и нажмите «Подключить». Папка используется на месте
  (не копируется). Хост покажет карточку проекта со списком её `.exe`.
* **Форма «Загрузить проект с GitHub»** — вставьте ссылку на публичный
  репозиторий (`https://github.com/owner/repo`) и нажмите «Загрузить». Хост
  скачает **весь** проект в папку `external-apps` и покажет карточку со
  списком `.exe`. ⚠ Код репозитория исполняется на этой машине — загружайте
  только доверенные проекты.
* **Карточка проекта** — название, метка источника (**локальный**/**github**)
  и сведения:
  * **версия** — у локального из README проекта, у github — ветка репозитория;
  * **последний путь** (локальный) — папка, которую вы указали; путь
    показывается даже если папка исчезла (карточка остаётся, запуск запрещён);
  * **репозиторий** (github) — источник загрузки;
  * **доступность** — живая проверка: у локального — «папка на месте», у
    github — отвечает ли исходный репозиторий (нужен интернет; проверка
    быстрая, ничего не скачивает);
  * **список всех найденных `.exe`** — у каждого файла поле аргументов и своя
    кнопка «Запустить».

  Кнопка «Удалить» отключает проект (github-проект удаляется с диска;
  локальный лишь отвязывается — ваша папка не трогается).
* **Кнопка «Запустить»** — запускает выбранный `.exe`. Можно нажимать
  несколько подряд: задания выполняются **параллельно** и появляются в
  таблице внизу.
* **Таблица заданий** — статус каждого запуска: `running` (можно «Отменить»),
  `succeeded` (в результате `{"exit_code":0,"stdout":"…"}`), `failed`,
  `cancelled`.
* **Кнопка «Обновить список»** — пересканирует проекты (после ручного
  изменения на диске).

## 4. Командная строка (необязательно)

Те же действия доступны без браузера:

    bin/scg-host                                     # локальный старт + браузер
    bin/scg-host projects                            # список проектов и их .exe
    bin/scg-host add-local C:\tools\my-app           # подключить локальный проект
    bin/scg-host download https://github.com/owner/repo   # скачать проект с GitHub
    bin/scg-host run-project my-app app.exe --arg --verbose   # запустить выбранный .exe
    bin/scg-host serve --addr :8080 --max-parallel 4 --open   # веб-режим

`run-project` печатает результат в JSON и завершается кодом 0 при успехе, 1
при ошибке — удобно для скриптов и CI. Каждый `--arg` добавляет один аргумент
запускаемой программе. Флаг `--dir` задаёт рабочий каталог (по умолчанию —
рядом с исполняемым файлом): там создаются `external-apps/` и `scg-apps.json`.

## 5. Как подключить проект

### 5.1. Локальный проект (папка на вашем компьютере)

Укажите путь к папке — на веб-странице (форма «Подключить локальный проект»)
или командой:

    bin/scg-host add-local /путь/к/папке

Хост запоминает путь и **использует папку на месте** (ничего не копирует),
показывает все найденные в ней `.exe`. Подключение постоянное: **последний
указанный путь** сохраняется в `scg-apps.json` и восстанавливается при
следующем запуске. Если папка исчезла (переместили, переименовали, диск
отключён) — карточка проекта **не пропадает**: она остаётся с последним
указанным путём и пометкой «папка недоступна», запуск при этом запрещён.
Верните папку на место и нажмите «Обновить список» — или подключите проект
заново с новым путём (повторное указание пути обновляет привязку).

**Версия** локального проекта берётся из его `README`
(`README.md`/`README.txt`/`README`): первая строка вида «Версия: 1.2.3»,
«version 1.2» или «v1.4». Если версии в README нет — показывается «—».

Удаление (кнопка «Удалить» или удалить строку из состояния) лишь отвязывает
проект — ваша исходная папка остаётся нетронутой.

### 5.2. Проект с GitHub (одна ссылка)

Вставьте ссылку на публичный репозиторий — на веб-странице (форма «Загрузить
проект с GitHub») или командой:

    bin/scg-host download https://github.com/owner/repo

Принимаются формы `owner/repo`, полный URL и `git@github.com:owner/repo`.

Что делает хост:

* скачивает архив репозитория (ветка `main`, если её нет — `master`);
* распаковывает **весь проект целиком** в подпапку папки `external-apps`
  (защита от zip-slip, exec-биты файлов сохраняются);
* пишет рядом `.scg-project.json` (ветка = версия, источник);
* показывает карточку проекта со списком всех найденных `.exe`.

**Версия** github-проекта — имя ветки. Загрузка постоянная: при каждом старте
папка `external-apps` сканируется. Повторная загрузка той же ссылки обновляет
проект. Удаление (кнопка «Удалить» или удалить подпапку в `external-apps`)
убирает проект с диска.

**Доступность репозитория** проверяется живьём (строка «Доступность» в
карточке и `GET /api/projects/{name}/availability`): хост делает быстрый
HEAD-запрос к тому же адресу, с которого проект скачивался, ничего не
загружая. «✓ репозиторий доступен» означает, что проект можно скачать
повторно; «✗ недоступен» — репозиторий удалён/переименован/стал приватным
или нет интернета (сам скачанный проект при этом продолжает работать).

Ограничения: нужен интернет; репозиторий должен быть публичным; `.exe` —
программы Windows (на хосте под Linux/macOS не запустятся); ⚠ запуск
скачанного `.exe` = исполнение чужого кода — только доверенные проекты.

### 5.3. Выбор и запуск `.exe`

Файлы `.exe` в карточке проекта упорядочены детерминированно: сперва
ближайшие к корню, затем по алфавиту. Выберите нужный, при необходимости
впишите аргументы (через пробел) и нажмите «Запустить». Результат появится
в таблице: `{"exit_code":0,"stdout":"…вывод программы…"}`. Ненулевой код
выхода → задание `failed` с кодом и stderr. Таймаут (по умолчанию 300 с) и
лимит вывода (16 МиБ) действуют всегда.

### 5.4. Что гарантирует хост (сепарация)

* каждый запуск — **новый процесс** операционной системы;
* рабочий каталог процесса — папка проекта;
* окружение минимально: только `PATH`, `HOME`, `LANG`, `LC_ALL`, `TMPDIR`,
  `SystemRoot`;
* таймаут принудительно завершает зависший процесс (в т.ч. его дочерние
  процессы на Windows — через `WaitDelay`);
* вывод ограничен 16 МиБ;
* отказ одного проекта не влияет на остальные.

## 6. Структура приложения: каждый файл и его роль

    scg-host/
    ├── go.mod                    имя модуля и версия Go; внешних зависимостей
    │                             НЕТ — только stdlib
    ├── Makefile                  короткие команды: build / test / serve
    ├── README.md                 этот документ
    │
    ├── cmd/scg-host/
    │   └── main.go               ТОЧКА ВХОДА. Без аргументов — локальный старт
    │                             с автооткрытием браузера. Подкоманды:
    │                             projects | add-local | download | run-project
    │                             | serve. Предметной логики нет.
    │
    ├── internal/host/            БИБЛИОТЕКА ХОСТА
    │   ├── ids.go                пакетный док, HostVersion (версия хоста) +
    │   │                         NewCallID (идентификатор задания)
    │   ├── launcher.go           Launcher.Run: запуск .exe (raw) в изоляции —
    │   │                         процесс, рабочий каталог, минимальное
    │   │                         окружение, таймаут (WaitDelay), лимит вывода;
    │   │                         resolveCommand ищет голое имя в папке проекта.
    │   ├── registry.go           Project (проект) и реестр двух источников:
    │   │                         local (по пути) + external (external-apps).
    │   │                         AddLocalProject, LoadState (восстановление
    │   │                         локальных путей — последний указанный путь
    │   │                         сохраняется даже для пропавшей папки),
    │   │                         Rescan + persistState (scg-apps.json),
    │   │                         RemoveProject, readmeVersion.
    │   ├── github.go             DownloadProject: скачивание архива
    │   │                         (main/master), безопасная распаковка ВСЕГО
    │   │                         проекта в external-apps, .scg-project.json,
    │   │                         findAllExes (список всех .exe),
    │   │                         CheckRepoAvailability (доступность репозитория).
    │   ├── jobs.go               Параллельность: Job и JobManager.StartProject
    │   │                         (запуск выбранного .exe через Launcher.Run;
    │   │                         недоступный проект отвергается), Cancel,
    │   │                         Wait, List/Get, семафор --max-parallel.
    │   ├── server.go             HTTP API: /api/projects (+ availability),
    │   │                         /api/jobs.
    │   ├── ui.go                 Встроенная веб-страница (html/template, без
    │   │                         CDN): формы, карточки с версией/путём/
    │   │                         доступностью, таблица заданий.
    │   ├── listen.go             Умный выбор порта (ListenSmart).
    │   └── host_test.go          Тесты (интеграционные — через sh, пропуск
    │                             без sh).
    │
    ├── external-apps/            папка скачанных github-проектов (создаётся
    │                             рядом с exe; в подпапке — .scg-project.json)
    └── scg-apps.json             сводка подключённых проектов; сохраняется
                                  и обновляется автоматически

### Как файлы взаимодействуют

Запуск: карточка проекта → `POST /api/projects/{name}/run` (**server.go**) →
**jobs.go** (`StartProject`: проверка, что .exe из списка проекта) →
**launcher.go** (`Run`: процесс, минимальное окружение) → результат
`{exit_code, stdout}` виден в таблице через `GET /api/jobs`.

Подключение локального: форма/`add-local` → **registry.go**
(`AddLocalProject`: запомнить путь) → persist. Загрузка github: форма/
`download` → **github.go** (`DownloadProject`) → **registry.go** (`Rescan`
находит проект и его .exe) → persist. При старте **main.go** (`openStore`)
восстанавливает локальные проекты из `scg-apps.json` (`LoadState`) и
сканирует `external-apps`.

Направление зависимостей одностороннее:
`main.go → server.go → jobs.go → launcher.go` и `registry.go → github.go`.
Приложения не зависят ни от чего — они не знают, что их вызывает этот хост.

## 7. HTTP API (для интеграций)

| Метод и путь | Назначение |
|---|---|
| `GET /api/projects` | список проектов (источник, версия, список .exe) |
| `POST /api/projects/rescan` | пересканировать локальные пути и external-apps |
| `POST /api/projects/local` | тело `{"dir":"/путь/к/папке"}` — подключить локальный проект |
| `POST /api/projects/github` | тело `{"repo":"ссылка"}` — скачать проект с GitHub |
| `POST /api/projects/{name}/run` | тело `{"executable":"bin/app.exe","args":["--flag"]}` — запустить .exe |
| `POST /api/projects/{name}/remove` | отключить проект (github — с удалением папки) |
| `GET /api/projects/{name}/availability` | живая проверка: local — папка на месте?; github — репозиторий отвечает? |
| `GET /api/jobs` | все задания, новые первыми |
| `GET /api/jobs/{id}` | статус и результат одного задания |
| `POST /api/jobs/{id}/cancel` | отмена выполняющегося задания |

## 8. Проверка и устранение неполадок

Запуск тестов: `go test ./...` (или `make test`).

| Симптом | Причина и действие |
|---|---|
| `go: command not found` | Go не установлен или не в PATH — повторите Шаг 1 |
| Проекта нет в списке | нажмите «Обновить список»; для локального — проверьте, что папка на месте |
| У проекта нет кнопок запуска | в папке не найдено ни одного `.exe` |
| Задание `failed: не удалось запустить` | файл не является исполняемым для этой ОС (например .exe под Windows на Linux), либо повреждён |
| Задание `failed: превышен таймаут` | программа работает дольше 300 с или ждёт ввода, которого нет |
| «bind: Only one usage of each socket address…» / порт занят | хост сам берёт следующий свободный порт; принудительно: `serve --addr :8090` |
| Браузер не открылся сам | откройте адрес из окна вручную; на Linux нужен `xdg-open` (пакет xdg-utils) |
| «репозиторий не найден или не публичный» | проверьте ссылку и что репозиторий публичный; пробуются ветки `main`, затем `master` |
| «GitHub недоступен (нужен интернет)» | загрузка репозитория — единственная функция, требующая интернета |
| Карточка локального проекта пунктирная, «папка недоступна» | папку переместили/удалили — карточка сохранена с последним указанным путём; верните папку и нажмите «Обновить список» или подключите заново |
| «Доступность: ✗ репозиторий недоступен» у github-проекта | репозиторий удалён/переименован/стал приватным или нет интернета; скачанная копия продолжает работать |
| Версия локального проекта не та | версия берётся из README (строка «Версия…»); поправьте README |

Хост не требует прав администратора; всё, кроме загрузки с GitHub, работает
в изолированной сети.
